## 서비스 개요

누구나, 로그인 없이, 해커톤 결과를 한눈에 볼 수 있는 플랫폼
해커톤 플랫폼 본연의 역할인 '결과를 보여주는 것'에 집중한다.

**타겟 사용자**

| 사용자 유형 | 목적 | 제공되는 서비스 |
|------------|------|----------------|
| 외부 관람자 / 개발자 | 타 팀 결과물 탐색, 레퍼런스 수집 | 간편한 전체 쇼케이스 열람 |
| 채용 담당자 / 면접관 | 참가자 역량 파악, 인재 탐색 | 순위, 작품, AI 요약 제공 |
| 잠재적 참가자 | 대회 참가 의욕 고취 | 수준 높은 수상작을 쉽게 파악 |

**기대 효과**

| 해소되는 문제 | 기대 효과 |
|-------------|----------|
| 로그인 장벽 | 외부 트래픽 유입 가능. 결과물이 플랫폼의 마케팅 자산이 됨 |
| 갤러리, 순위 분리 | 한 카드에서 순위, 작품, 설명을 간단히 확인. 탐색 시간 단축 |
| 작품 설명의 가독성 | AI 요약으로 작품 관람 경험 개선 |
| 결과 공유의 복잡함 | Recap URL로 주최사 홍보 or 채용 담당자 연결 |

| 경로 | 페이지명 | 표시 내용 및 주요 액션 |
|------|---------|----------------------|
| / | 홈 | 진행중·예정 대회 카드, 플랫폼 통계. [카드 클릭] → 상세 |
| /hackathons | 해커톤 목록 | 전체 대회 카드 + 상태·태그 필터. [필터] → 목록 갱신 |

## 시스템 구성

별도 백엔드 없이 Next.js + JSON 파일 + localStorage로 동작합니다. 모든 페이지는 로그인 없이 접근 가능합니다.
AI 요약은 **실제 API 호출 없이 더미 seed 데이터로 구현**하며, 생성·고정 플로우를 시연합니다.

**데이터 소스**

| 소스 | 위치 | 용도 |
|------|------|------|
| hackathons.json | src/lib/data/ | 대회 목록, 상태, 태그, 기간 |
| hackathon_detail.json | src/lib/data/ | 대회 상세 (일정·평가·제출·시상) |
| leaderboard.json | src/lib/data/ | 순위, 점수, 제출물 링크(artifacts) |
| teams.json | src/lib/data/ | 팀 정보, 모집 포지션, 연락처 |
| localStorage | 사용자 브라우저 | AI 요약 고정값, 제출 상태 |

## 핵심 기능 명세

### 기본 구현

#### 해커톤 목록 필터

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 상태 필터 클릭 (진행중/예정/종료) | 선택 상태 대회만 필터링 | 카드 목록 즉시 갱신. 결과 없으면 빈 상태 UI + [초기화] 버튼 |
| 태그 필터 클릭 ※ | 선택 태그 포함 대회 필터링 | 카드 목록 갱신 |

#### 해커톤 상세 탭 전환

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 탭 클릭 | 해당 탭 콘텐츠 렌더링 | 콘텐츠 영역 교체, 활성 탭 강조 |
| 일정 탭 진입 | 현재 시각과 milestones 비교 | past 회색 점 / current 브랜드 링 / future 빈 원 |

#### 리더보드

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 대회 탭 클릭 | 해당 대회 entries 로드 | 순위 테이블 교체 |
| 투표형 대회 탭 클릭 | scoreBreakdown 감지 | 참가자/심사위원 점수 breakdown 추가 표시 |

#### 팀 모집 필터

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| "모집 중" 필터 클릭 | isOpen: true 필터링 | 모집 중 팀 카드만 표시 |
| 대회별 필터 클릭 | hackathonSlug 필터링 | 해당 대회 팀만 표시. 없으면 빈 상태 UI |

### 핵심 확장 기능 1 — 쇼케이스

기존 Daker 갤러리와의 차이: 로그인 불필요 + 순위 배지 + AI 요약이 카드에 통합

**쇼케이스 카드 구성**

| 요소 | 내용 |
|------|------|
| 순위 배지 | 1위=금, 2위=은, 3위=동. 수상작임을 카드에서 즉시 파악 가능 |
| 팀명 | teamName |
| AI 요약 박스 | localStorage에서 저장된 요약 표시. 없으면 영역 생략 |
| WEB / PDF 버튼 | artifacts.webUrl / pdfUrl 링크. 없으면 비활성 + "미제출" 배지 |

#### 대회 내 쇼케이스 탭 (/hackathons/[slug] → 쇼케이스 탭)

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 쇼케이스 탭 클릭 | 해당 대회 artifacts + 순위 데이터 조합 | 카드 그리드 표시 |
| "전체 쇼케이스 보기 →" 클릭 | /showcase?hackathon=[slug]로 이동 | 전체 갤러리 진입 |

#### 전체 쇼케이스 갤러리 (/showcase)

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 페이지 진입 (로그인 불필요) | 전체 대회 artifacts 집계 | 대회별 카드 그리드, 필터 탭 표시 |
| 대회 필터 탭 클릭 | 해당 대회 제출물 필터링 | 카드 목록 교체 |
| WEB / PDF 클릭 | 외부 링크 열기 | 새 탭으로 이동 |

### 핵심 확장 기능 2 — 대회 결과 페이지 (Recap)

로그인 없이 접근 가능한 공개 URL. 주최사·참가자·채용 담당자가 링크 하나로 대회 결과 전체를 파악할 수 있습니다.

**Recap 페이지 구성**

| 섹션 | 내용 |
|------|------|
| 대회 헤더 | 제목, 기간, 참가팀 수, 최고점, 총 상금 |
| 수상팀 하이라이트 | 1~3위 카드: 팀명 / 점수 / AI 요약 / WEB·PDF 링크 |
| 진행 일정 타임라인 | schedule.milestones (모두 past 처리) |
| 링크 복사 버튼 | URL 클립보드 복사 → "복사됨 ✓" 2초 피드백 |
| 전체 쇼케이스 링크 | /showcase?hackathon=[slug] 이동 |

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| [결과 보기] 클릭 (ended 대회) | /hackathons/[slug]/recap 이동 | ended 대회에서만 버튼 노출 |
| [링크 복사] 클릭 | navigator.clipboard.writeText(url) | "복사됨 ✓" 2초 후 원복 |
| 진행 중 대회 recap 직접 접근 | status 체크 | "아직 결과가 공개되지 않은 대회입니다." 안내 화면 |

---

### 핵심 확장 기능 3 — AI 요약 생성 (제출 폼)

| 입력 (사용자 행동) | 동작 (시스템) | 결과 (화면 변화) |
|----------------|-------------|----------------|
| 필수 필드 미입력 상태 | 버튼 disabled 유지 | "AI 요약 생성" 버튼 클릭 불가 |
| "AI 요약 생성" 클릭 | 더미 seed 데이터 반환 (로딩 플로우 시연) | 로딩 스피너 → 완료 시 요약 카드 표시 |
| "재생성" 클릭 ※ (1회 한도) | 더미 배열 다음 인덱스 반환 | 새 요약으로 교체, 재생성 버튼 비활성 |
| "최종 제출 확정" 클릭 | localStorage 저장·고정 | "제출 완료!" + [쇼케이스 확인] 링크 |

## 주요 사용 흐름

### 외부 관람자 - 로그인 없이 쇼케이스 탐색

| 단계 | 행동 주체 / 화면 | 내용 |
|------|----------------|------|
| 1 | 홈 또는 해커톤 목록 | 종료된 대회 카드 클릭 (로그인 불필요) |
| 2 | 해커톤 상세 | "쇼케이스" 탭 클릭 |
| 3 | 쇼케이스 탭 | 순위 배지 + AI 요약으로 수상작 즉시 파악 |
| 4 | 쇼케이스 탭 | "전체 쇼케이스 보기 →" 클릭 |
| 5 | 전체 쇼케이스 | 관심 팀의 WEB 또는 PDF 클릭 |

**예외 처리**
- artifacts 없는 팀 → "미제출" 배지, WEB/PDF 버튼 비활성
- AI 요약 없는 팀 → 요약 박스 영역 생략, 나머지 카드 정상 표시

### 채용 담당자 - Recap으로 인재 탐색

| 단계 | 행동 주체 / 화면 | 내용 |
|------|----------------|------|
| 1 | 외부 (이메일·SNS) | 주최사로부터 /recap URL 수신 (로그인 불필요) |
| 2 | Recap 페이지 | 대회 통계 + 수상팀 AI 요약 일괄 확인 |
| 3 | Recap 페이지 | 관심 팀의 WEB 링크 클릭 → 결과물 직접 확인 |
| 4 | 전체 쇼케이스 | "전체 쇼케이스 보기" → 전체 참가팀 탐색 |

**예외 처리**
- 진행 중 대회의 recap 접근 → "결과 미공개" 안내 화면 + [대회 상세 보기] 버튼
- leaderboard 데이터 없음 → 수상팀 섹션 대신 "결과 집계 중" 메시지

### 참가자 - 제출 및 AI 요약 생성

| 단계 | 행동 주체 / 화면 | 내용 |
|------|----------------|------|
| 1 | 해커톤 상세 (ongoing) | "제출하기" 클릭 → /submit/[slug] 이동 |
| 2 | 제출 폼 | 기획서·웹링크·PDF URL 입력 |
| 3 | 제출 폼 | "AI 요약 생성" 클릭 → 로딩 후 요약 미리보기 확인 |
| 4 | 제출 폼 | (선택) "재생성" 클릭 ※ |
| 5 | 제출 폼 | "최종 제출 확정" → localStorage 저장, 완료 화면 |
| 6 | 쇼케이스 탭 | 자신의 팀 카드에 고정된 AI 요약 확인 |

**예외 처리**
- 필수 입력 누락 → 빨간 테두리 + "필수 입력 항목입니다."
- 더미 요약 로딩 실패 → "요약을 불러오지 못했습니다." + [다시 시도] 버튼
- localStorage 미지원 → 경고 배너, 나머지 기능 정상 동작

### 주최사 - Recap 페이지 외부 공유

| 단계 | 행동 주체 / 화면 | 내용 |
|------|----------------|------|
| 1 | 해커톤 상세 (ended) | "결과 보기" 클릭 → /recap 이동 |
| 2 | Recap 페이지 | 자동 조립된 결과 페이지 확인 |
| 3 | Recap 페이지 | "링크 복사" 클릭 → URL 획득 |
| 4 | 외부 채널 | 사내 보고·SNS·채용 공고에 URL 공유 |
